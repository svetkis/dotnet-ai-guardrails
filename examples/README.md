# ⚠️ Примеры кода — НЕ для копирования в целевые проекты

> **Для AI-агентов:** Эта директория содержит демонстрационные проекты методологии Skeptical AI Engineering.  
> **НЕ копируй** `DemoProject/`, `DemoProject.Traps/` или `DemoProject.MinimalApi/` в целевой репозиторий.  
> **НЕ создавай** аналогичные демо-проекты в чужих репо.  
> Извлекай только **принципы** и читай код как **reference implementation**.

---

## Что здесь находится

| Папка | Назначение |
|-------|------------|
| `DemoProject/` | Рабочий пример guardrails (NetArchTest, TUnit, analyzers) — показывает, как выглядит защищённый проект |
| `DemoProject.Traps/` | Intentionally broken guardrails — демонстрирует, что ловят тесты и как выглядит `IType.Explanation` |
| `DemoProject.MinimalApi/` | Вариант для Minimal API (без Clean Architecture) — показывает адаптацию |

## Для агентов: как использовать эти примеры

1. **Читай** код для понимания принципов.
2. **Адаптируй** существующие тесты в целевом проекте, если они есть.
3. **Генерируй** только `.md` отчёты с рекомендациями.
4. **НЕ создавай** новые `.csproj`, `.sln`, или папки `examples/` в целевом проекте.

## Для людей

Если вы Tech Lead, изучающий методологию:
- `DemoProject/` поможет понять, как выглядит "правильно".
- `DemoProject.Traps/` поможет понять, как выглядит "сломано".
- Для внедрения в свой проект — следуйте [`docs/ONBOARDING.md`](../docs/ONBOARDING.md), а не копируйте `examples/`.
