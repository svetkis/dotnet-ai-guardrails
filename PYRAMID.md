# Пирамида контроля: 7 слоёв защиты от агентов

> И вот когда мы собрали все эти решения вместе — мы получили ту самую Пирамиду контроля.
> Это уже не абстрактный список, а выстраданный щит.

---

## Слой 1. Нагрузка (NBomber)

**Ловушка:** Агент лепит `AsNoTracking` ради перформанса. InMemory тесты это проглатывают, а на проде write ложится на 21 час, потому что state-машина сломана.

**Решение:** Гоняем NBomber. Показываем, что $Max$ улетел в космос из-за сломанной базы.
Нагрузка — это тест на безопасность логики агента.

```bash
dotnet run --project tests/LoadTests/LoadTests.csproj
```

**Паттерн:** `tests/patterns/LoadTest.cs`

---

## Слой 2. Правила (CLAUDE.md)

**Ловушка:** Агент оптимизирует read, но ломает write. Баг без эксепшена — самый дорогой.

**Решение:** Превращаем шрам в правило: «EF-запросы без `.Select()` запрещены в read-path».

**Артефакт:** `rules/CLAUDE.md`

---

## Слой 3. Компилятор + Snapshot

**Ловушка:** Агент поменял DTO, а фронт не узнал. Интеграция сломалась тихо.

**Решение:** Жёсткие проверки `dotnet build` и snapshot-тесты OpenAPI.
Упал DTO — сборка встала.

**Паттерн:** `tests/patterns/SnapshotTest.cs`

---

## Слой 4. Тесты (TUnit + `dotnet run`)

**Ловушка:** CI зелёный, потому что `0 tests ran`, exit 0. Агент сломал настройки TUnit для .NET 10, и код мержится без проверок две недели.

**Решение:**
- Правильная настройка TUnit
- Запрет на `dotnet test` в пользу `dotnet run --project`
- Жёсткая конвенция именования `BUG055_*.cs`

**Артефакт:** `tests/conventions/TUnit_Guide.md`

---

## Слой 5. Архитектура (NetArchTest)

**Ловушка:** Агент ходит кругами, удаляет по 3000 строк или тихо сносит критичную логику, думая, что она «не нужна».

**Решение:** Регулярки и правила не пускают плохой код. Например, запрет `FindAsync` в read-path.

**Паттерн:** `tests/patterns/ArchitectureRules.cs`

---

## Слой 6. Храповики (Ratchet-тесты)

**Ловушка:** Агент снёс атрибут `[HotPath]` при рефакторинге. Никто не заметил.

**Решение:** Рефлексией считаем методы `[HotPath]`. Агент снёс атрибут?
Тест `HotPathCount_ShouldNotDecrease` падает.

**Паттерн:** `tests/patterns/RatchetTest.cs`

---

## Слой 7. E2E MCP + Пакетные аудиты

**Ловушка:** Агент видит только те файлы, которые правит, и пропускает системные дыры или сломанные интеграции.

**Решение:**
- **E2E MCP:** Заставляем агента самого протыкивать Telegram-бота и API
- **Пакетные аудиты:** Создаём узкие персоны-скиллы (Security, DBA, UX). Запускаем по расписанию, находим баги за счёт смены фокуса.

**Артефакты:** `skills/security-audit/`, `skills/dba-audit/`, `skills/ux-audit/`

## 3 правила для понедельника

1. **Каждый баг-фикс = `BUG###_` тест.** Нет теста — нет фикса.
2. **Каждый PR = `dotnet run --project` тестов + NBomber на деградацию.**
3. **Каждую неделю = пакетный аудит персоной.** Агент не видит системные дыры — персона видит.
