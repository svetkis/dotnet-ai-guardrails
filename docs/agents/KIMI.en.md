# Kimi Code CLI — Guardrails Integration

> Kimi Code CLI uses the skill system in `.kimi/skills/`.
> This is the native format for the `dotnet-skeptical-ai` repository.

## Integration Structure

```
.kimi/
└── skills/
    ├── skeptical-ai-bootstrap/          # Scanning + backlog
    ├── code-review/                 # Review on every PR
    ├── task-compliance/             # Scope check
    ├── security-audit/              # Security audit
    ├── dba-audit/                   # DB audit
    ├── performance-audit/           # Performance audit
    ├── api-design-audit/            # API design audit
    ├── bot-audit/                   # Telegram bot audit
    ├── i18n-audit/                  # Localization audit
    └── {project-specific}/          # Custom skills
```

## Project Configuration

Create `.kimi/skills/README.md` (generated by onboarding):
- Map of all skills
- Statuses Active / WIP / Backlog
- Integration diagram

## Running Onboarding

```bash
# Install the onboarding skill
cp -r /path/to/dotnet-skeptical-ai/templates/skills/skeptical-ai-bootstrap ./.kimi/skills/

# Run
kimi run skeptical-ai-bootstrap
```

## Nuances

- Skills are Markdown files with YAML frontmatter
- The agent reads `SKILL.md` and applies the instructions
- Custom skills are created from the `NEW-SKILL-TEMPLATE.md` template
- CI integration via `kimi run {skill-name}` or manual launch

## Limitations

- Kimi must be installed locally (`kimi` CLI)
- Skills do not auto-launch on PR — a CI wrapper or manual launch is needed
- Context is limited by the model window (~200k tokens)
