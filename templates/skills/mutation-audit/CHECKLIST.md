# Mutation Audit — Чеклист

## Перед началом
- [ ] Установлен `dotnet-stryker` (глобально или локально)
- [ ] Определены 1–3 критичные сборки
- [ ] Известен бюджет времени на прогон

## Конфигурация
- [ ] `stryker-config.json` создан для каждой сборки
- [ ] `dotnet-tools.json` содержит `dotnet-stryker`
- [ ] Конфигурация совместима с тестовым фреймворком проекта

## Baseline и ratchet
- [ ] Mutation score baseline зафиксирован
- [ ] `MutationGuardTest` добавлен в тестовый проект
- [ ] Минимальный порог определён (например, 70%)

## Анализ мутантов
- [ ] Топ выживших мутантов разобран
- [ ] Для каждого добавлен/усилен тест
- [ ] False-positive (эквивалентные) мутанты задокументированы

## Интеграция
- [ ] Stryker запускается перед релизом / раз в спринт
- [ ] Отчёт сохраняется как артефакт CI
- [ ] Не блокирует обычный PR

## Формат отчёта
- [ ] Сводка по сборкам
- [ ] Топ выживших мутантов
- [ ] False positives
